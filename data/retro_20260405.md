## Bugs Found & Fixed During Run

### P0 — Dialogue Event Stuck Loop (fixed mid-run)
Events with dialogue text (e.g. "Fan Letter", "Mystery Fortune Ritual") were detected as `event` screen type because they show "Trainee Event" header. But they have no choice buttons — only Skip/Quick/Log at the bottom. The bot tapped center (540, 960) to dismiss, which hit the character sprite and did nothing. Got stuck for 12+ iterations.

**Fix:** When no event choices found and Skip button is visible in OCR, tap dialogue area (540, 1500) instead of center. Also added stuck loop breaker — bail out after 15 consecutive identical results in `run_one_turn()`.

### P0 — Reset Whistle Infinite Retry Loop (fixed mid-run)
During TS Climax, the `_summer_whistle_used` flag was reset every time the `ts_climax_home` handler ran. When the whistle couldn't be found in Training Items (scroll bug), the bot would: try whistle → fail → return to ts_climax_home → reset flag → try whistle again → infinite loop.

**Fix:** Removed `_summer_whistle_used = False` from `ts_climax_home` handler. Also added inventory cleanup on failure — remove phantom items from tracked inventory when `_use_training_items()` fails.

### P1 — Skip Button Toggled Off by Unknown Handler (fixed mid-run)
The unknown screen handler searched for green buttons at y=1600-1900, x=300-950. The Skip button (green, ~x=397, y=1870) fell in this range. Bot tapped it, toggling Skip OFF, which caused dialogue events to show full text instead of auto-advancing.

**Fix:** Narrowed green button x-range to (460, 950) in the unknown handler, skipping past Skip/Quick buttons.

### P1 — Phantom Inventory Items (fixed mid-run)
When `_use_training_items()` failed to find an item (e.g. `empowering_mega` that was miscategorized from shop), it left the phantom item in the shop manager's inventory. Next attempt would try the phantom item again instead of falling back to alternatives (e.g. `motivating_mega`).

**Fix:** On item use failure, decrement and remove the item from tracked inventory. Applied to megaphone, ankle weight, and reset whistle failure paths.

## Training Items Scroll Bug (Remaining)

The most impactful remaining bug. When the inventory has 7+ items, some items are below the fold. The scroll function (`swipe 540,1350→540,750`) either:
1. Doesn't actually scroll the list, or
2. Scrolls but the OCR item-name y-positions don't align with the green + button y-positions after scrolling

This caused failures for: Reset Whistle, Empowering Megaphone, Motivating Megaphone, and Manuals during TS Climax. Items on page 0 (Vita, Royal Kale) work fine. Items that require scrolling frequently fail.
